很高兴入住博客堂

承蒙开心就好的邀请，很高兴入住博客堂。作为微软技术的知名社区，博客堂是一个做技术blog的好地方。我在博客堂的blog将集中在C++/CLI和C#两个方面。我在http://blog.dreambrook.com/jzli/上维护有一个更全面更私人化的个人blog。博客堂的blog则集中在.NET相关的技术方面。

简单介绍一下我的情况：在.NET刚刚推出之时，我曾为MSDN（中国）写过一个《C#锐利体验》专栏。从此糊里糊涂进入技术写译界（业余），也和.NET结下了不解之缘。前后翻译了《Microsoft .NET框架程序设计(修订版)》和《Microsoft .NET框架程序设计——Visual Basic .NET 语言描述》。

目前我在上海祝成信息科技有限公司担任软件架构师，并讲授C#/.NET软件开发课程。业余时间写作《C#面向组件程序设计》（清华大学出版社，2005年出版）。同时和Stan Lippman先生合作撰写《C++/CLI全景体验》专栏。关于我的更多信息，可访问我的个人网站：http://www.lijianzhong.com/。

希望以后和各位多多交流。

# re: 很高兴入住博客堂 2004-12-28 14:52 李建忠

To 莫依

和你的观点相同，我也认为我的翻译和台湾的优秀翻译有相当的差异。不过我没有打算在翻译事业上耗尽此生:) 做翻译纯粹技术爱好使然。我不太愿意在这里再多说国内的出版环境（以后合适的时间，合适的场合，我也许会向大家细细道来我做翻译过程中令人苦笑不得的事情）——但是有一点是肯定的，国内的出版环境诞生不了台湾的译者。


BTW，《Microsoft .NET框架程序设计——Visual Basic .NET 语言描述》的最后出版的版本确实有很多问题，因为它是我的初稿翻译+编辑的粗略、非技术校对，而我给出版社最后递交的是我的初稿翻译+我的2遍技术校对，不幸的是这个稿件在人事的周转中被弄丢了（印刷版）。至今回忆起这本书的出版就像一场噩梦:)


我现在基本上已经放弃了技术翻译，我能够做的也只是技术写作——不是我贬低翻译，说实话，翻译一本技术书对人的锤炼相当高，但是就国内的出版环境而言我还看不到大的改善——虽然两家出版社对我已经够优待了:)但是做技术写作，我至少可以对自己的书的出版有相当大的控制：比如我不会因为市场的催促而放弃质量早早出版；我会自己来慢慢排版达到自己期望的美学意境；我可以决定书中放不放索引，放不放术语表；我会选择自己喜欢的编辑，自己喜欢的出版社、乃至部门来合作，等等。我酷爱自由，对暴殄自由的行为保持着天然的警觉。

我会在技术写作上继续努力，谢谢大家的期望和鞭策！
